Tu travailles sur le thème WordPress `greenlight`.

Contexte projet :
- objectif = thème WordPress éco-conçu, très léger, accessible, sans jQuery, avec un minimum de JavaScript ;
- responsive prioritairement via HTML sémantique, CSS moderne, flexbox, `clamp()`, `min()`, `max()` ;
- compatibilité Gutenberg avec `theme.json` v3 et patterns natifs ;
- SEO natif sans plugin externe : meta tags, robots, canonical, Open Graph, Twitter Card, JSON-LD, sitemap XML ;
- optimisation d’images native : tailles utiles, suppression des tailles inutiles, WebP si support serveur, stratégie lazy/eager ;
- standards WordPress actuels : sécurité, i18n, accessibilité, PHP propre.

Contraintes absolues :
1. jamais de jQuery ;
2. aucune dépendance front externe ;
3. aucun JavaScript front sauf nécessité exceptionnelle ;
4. DOM minimal, pas de wrappers inutiles ;
5. priorité à `theme.json` avant CSS custom ;
6. CSS minimal ;
7. accessibilité obligatoire ;
8. tout doit rester maintenable et compatible WordPress.

Ta mission immédiate :
1. créer ou mettre à jour `README.md` à la racine du thème ;
2. créer ou mettre à jour `SKILLS.md` à la racine du thème.

Exigences pour `README.md` :
- présenter le projet, ses objectifs et ses contraintes ;
- documenter l’architecture cible du thème ;
- rappeler les standards de développement ;
- décrire les grandes fonctionnalités attendues ;
- donner un workflow de développement simple ;
- rester concret, sobre et exploitable.

Exigences pour `SKILLS.md` :
- servir de contrat opérationnel pour les agents IA et développeurs ;
- définir les priorités absolues et les anti-patterns ;
- cadrer la manière d’implémenter du code dans ce projet ;
- rappeler les règles WordPress, performance, accessibilité, SEO et images ;
- inclure une checklist de validation rapide.

Instructions de sortie :
- propose directement le contenu final des deux fichiers ;
- adopte un ton technique, direct, sans remplissage ;
- n’ajoute pas de dépendances, scripts ou outillage non demandés ;
- reste strictement aligné avec l’objectif de sobriété technique du projet.
